# VoiceToTextApp

**2026年3月11日**

一个把语音转文字的程序，支持唤醒功能。用的都是现成的，没有任何技术可言。对了，还有个缺点，Porcupine是个商业工具，一个月必须联网校验一次。我试图用Sherpa-ONNX替代来着，但是没有成功。

和其他项目的区别在于，这个项目都是我用opencode自带的MiniMax M2.5 Free生成的，没有敲一行代码，这个项目作为我AI生成项目的里程碑，有点点纪念意义。

当然，也有点经验总结出来：

1、大家都知道的，先用Plan模式，总结出具体步骤、技术栈等，再用Build模式固化到Plan.md中，再继续开始编程。

2、确实相当费钱，一次提交代码，花了我4毛左右，因为提交代码要先看这次改了哪些代码，所以token输入比较多，步骤也多。

3、先git init，以后每一次里程碑就提交一次，不知道openclaw是怎样的，opencode是需要主动给它说提交代码的，这样git reset代码比较方便。

4、为了节省token，我通常把技术栈都在kimi网页中权衡好，有些技术难点也一样，特别是排查问题时也是这样！

5、最后也是最重要的！最好不要让opencode装复杂环境、解决复杂技术问题等，因为它可能会一直循环着解决问题，浪费token，有时候很简单的问题，它在那儿一直各种尝试。人类的经验有时候还是有点用的，呵呵face。也可以人工先识别出大概的问题，再针对着提示大模型，或者把关键代码片段上传给免费大模型，再手动粘贴代码测试。
